    Introdução

    Bem-vindo a API de gerencialmente de Usuários. Aqui estão disponíveis todas as informações técnicas necessárias para a integração das aplicações clientes com esse serviço.

    Usuários

    Obter uma lista de usuários

    curl "http://ec2-34-229-214-99.compute-1.amazonaws.com:3000/users"

    A requisição acima retorna um array com todas os usuários registrados. A estrutura do JSON é exemplificada abaixo:

    [
    {
        "id": 1,
        "name": "Lucas Santos",
        "email": "lucas.santos.silva2011@gmail.com",
        "birthdate": "1997-01-11",
        "phone_number": "11942890139",
        "created_at": "2018-03-15T18:27:59.211Z",
        "updated_at": "2018-03-15T18:27:59.211Z"
    }
]

    Esse endpoint retorna todas os usuários gravados no banco de dados

    Requisição HTTP

    GET http://ec2-34-229-214-99.compute-1.amazonaws.com:3000/users

    Obter um usuário específico

    curl "http://ec2-34-229-214-99.compute-1.amazonaws.com:3000/users/<ID>"

    A requisição acima retorna um registro com uma estrutura em JSON, como exemplificado abaixo:

    {
    "id": 1,
    "name": "Lucas Santos",
    "email": "lucas.santos.silva2011@gmail.com",
    "birthdate": "1997-01-11",
    "phone_number": "11942890139",
    "created_at": "2018-03-15T18:27:59.211Z",
    "updated_at": "2018-03-15T18:27:59.211Z"
}

    Esse endpoint retorna um usuário específico.

    Requisição HTTP

    GET http://ec2-34-229-214-99.compute-1.amazonaws.com:3000/users/<ID>

    Parâmetros na URL

    Parâmetro Desccrição
    ID ID do usuário

    Criar um usuário

    curl "http://ec2-34-229-214-99.compute-1.amazonaws.com:3000/users"
  -X POST

    Para criar um novo usuário é necessário enviar no corpo da requisição uma estrutura em JSON estruturado como exemplificado abaixo:

    {
    "user": {
        "name": "Lucas Santos",
        "email": "lucas.santos.silva2011@gmail.com",
        "birthdate": "1997-01-11",
        "phone_number": "11942890139"
    }
}

    O retorno será um JSON contendo os dados gravados, jundo com o ID gerado:

    {
    "id": 1,
    "name": "Lucas Santos",
    "email": "lucas.santos.silva2011@gmail.com",
    "birthdate": "1997-01-11",
    "phone_number": "11942890139",
    "created_at": "2018-03-15T18:27:59.211Z",
    "updated_at": "2018-03-15T18:27:59.211Z"
}

    Esse endpoint cria um registro de um usuário

    Requisição HTTP

    POST http://ec2-34-229-214-99.compute-1.amazonaws.com:3000/users

    Parâmetros na URL

    Parâmetro Descrição

    Atualizar dados de um usuário

    curl "http://ec2-34-229-214-99.compute-1.amazonaws.com:3000/users/<ID>"
  -X PUT

    Para atualizar os dados de um usuário é necessário enviar no corpo da requisição uma estrutura em JSON, contendo os valores dos campos que serão atualizados.

    {
    "user": {
        "name": "Lucas Santos",
        "email": "lucas.santos.silva2011@gmail.com",
        "birthdate": "1997-01-11",
        "phone_number": "11942890139"
    }
}

    O retorno será um JSON contendo o registro do usuário, com os dados atualizados:

    {
    "id": 1,
    "name": "Lucas Santos",
    "email": "lucas.santos.silva2011@gmail.com",
    "birthdate": "1997-01-11",
    "phone_number": "11942890139",
    "created_at": "2018-03-15T18:27:59.211Z",
    "updated_at": "2018-03-15T18:27:59.211Z"
}

    Esse endpoint atualiza dados de um registro de usuário

    Requisição HTTP

    PUT http://ec2-34-229-214-99.compute-1.amazonaws.com:3000/users/<ID>

    Parâmetros na URL

    Parâmetro Descrição
    ID ID do usuário

    Erros

    A API do Admcorp utiliza os seguintes códigos de erros:

    Código Significado
    400 Bad Request -- Houve um erro na requisição.
    401 Unauthorized -- Chave da autenticação não foi encontrada ou é inválida.
    403 Forbidden -- Acesso não autorizado.
    404 Not Found -- O recurso não pode ser encontrado.
    405 Method Not Allowed -- Tentativa de acesso à um recurso com método invalido.
    406 Not Acceptable -- O formato requisitado é diferente de JSON.
    500 Internal Server Error -- Problema interno com a aplicação ou servidor.
    503 Service Unavailable -- API offline.
